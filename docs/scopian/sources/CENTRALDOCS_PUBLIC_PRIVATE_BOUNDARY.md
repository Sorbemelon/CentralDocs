# CentralDocs Public/Private Repo Boundary

## Public and pushed

- `frontend/`
- `backend/`
- `backend/mock-data/`
- `docs/scopian/sources/`
- `README.md`
- `.gitignore`
- `.crosshelixignore`
- `.env.example` files

## Private/local and ignored

- `_reference/`
- `docs/scopian/private/`
- `docs/prompts/`
- `docs/build-progress/`
- `.agent-work/`
- `.env` and `.env.*`
- local uploads/cache/logs

## Reason

Public repo should show a professional portfolio project. It should not include build prompts, agent scratch work, progress logs, private credentials, or old messy reference repositories.
